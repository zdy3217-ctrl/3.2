# NarratorAI Omega 自动开发工作流

> **本文件可随时编辑！** 改完告诉AI「我改了工作流文件」即可。

---

## 一、触发方式

```
我是你爸爸
```

## 二、当前阶段：全面审查模式（禁止新增功能）

### ⚠️ 铁律：不准新增任何功能！只做审查、修复、清理！

### 工作节奏
1. ⚠️🔴 **每次触发"我是你爸爸"，只派出1个sub-agent** 🔴⚠️
2. ⚠️🔴 **严格串行！必须等上一个sub-agent完成后才能派下一个！绝不并发！** 🔴⚠️
3. ⚠️🔴 **sub-agent完成后立刻自动派下一个，不需要主人确认！直到主人说「停」才停！** 🔴⚠️
4. ⚠️🔴 **派出前必须用subagents list检查，有活跃的就等，不能派！** 🔴⚠️
5. 每轮审查一批文件（按目录分批），修复所有问题
6. 每轮完成后必须在钉钉主动汇报进度给主人
7. ⚠️ **每轮开始时发消息：「🔍 第N轮审查开始（审查范围：XXX）」** ⚠️
8. ⚠️ **每轮结束时发详细消息：修复了什么、删除了什么、还剩什么问题」** ⚠️

---

## 三、主助手（包工头）职责

1. 读取本文件
2. **派出前先 subagents list 检查，有活跃的就等！**
3. 用 `sessions_spawn` 派出 sub-agent 干活（用Opus 4.6模型）
4. sub-agent 完成后，在钉钉告诉主人详细修了什么
5. **确认无活跃sub-agent后**，自动派下一个
6. 无限接力，直到主人说「停」或零错误

### 汇报格式（每轮结束时）

```
✅ 第N轮审查完成（范围：XXX）

修复问题：
1. 修复了XXX — 文件路径
2. 删除了XXX — 原因
...

剩余问题：X个
自动派出第N+1轮审查...
```

---

## 四、主人控制命令

| 命令 | 作用 |
|------|------|
| `回到第N次` | git revert 回滚到第N个 commit |
| `撤回` | git revert HEAD |
| `停` | 停止派出新 sub-agent |
| `继续` | 恢复派出 sub-agent |
| `改方向` | 主人说新需求方向 |
| `我改了工作流文件` | 重新读取本文件 |

## 五、代码规范

### 技术栈
- Next.js 16 + TypeScript + Tailwind CSS
- 状态管理：Zustand（src/stores/projectStore.ts）
- UI风格：可爱简约风（奶白+马卡龙+胶囊Tab+玻璃拟态）

### UI风格规范
- 背景色：`bg-[#faf8f5]` 或 `bg-[#FFF8F0]`
- 卡片：`bg-white/70 backdrop-blur border border-gray-200/60 rounded-2xl`
- 主色渐变：`from-purple-500 to-pink-500`
- 按钮：圆角胶囊 `rounded-full` 或 `rounded-xl`
- 文字：标题 `text-gray-800`，正文 `text-gray-600`，辅助 `text-gray-400`
